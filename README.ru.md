## 1. Авторское право и лицензия

**Название ПО:** Chayuan AI Document Assistant (китайское официальное имя продукта: **察元 AI 文档助手**, npm-пакет **`chayuan`**).

Исходный код распространяется на условиях **[Apache License, Version 2.0](LICENSE)**. При соблюдении лицензии допускаются использование, изменение, публикация и **коммерческое использование**. Отдельный письменный договор (коммерческая лицензия, OEM и т. п.) имеет **преимущественную силу**, где это предусмотрено.

**Правообладатель / издатель:** Beijing Zhilingniao Technology Center (北京智灵鸟科技中心). **Сайт:** [https://aidooo.com](https://aidooo.com).

**Отказ от ответственности (кратко):** ПО предоставляется «как есть». Вывод моделей может быть неточным. Проверки конфиденциальности и «следов ИИ» носят **вспомогательный** характер и не являются юридическим или экспертным заключением.

## 2. Особое условие: не изменять брендинг «察元» в интерфейсе

**Без письменного разрешения** правообладателя нельзя заменять, удалять, скрывать или вводить в заблуждение относительно происхождения ПО, изменяя в пользовательском интерфейсе фиксированные строки бренда **察元** и связанные названия (например **察元 AI**, **察元 AI 文档助手**, пункты вроде **关于察元**, **添加到察元 AI 助手**, подписи типов файлов **察元模板** / **察元规则** / **察元文档** и т. д.) в:

- ленте **Ribbon** WPS Writer;
- **контекстном меню**;
- **диалогах, панелях задач, странице «О программе», приветствии**.

Изменение исходного кода внутри организации по Apache 2.0 не запрещено; при **распространении сборок** конечным пользователям с официальным UI нужно **сохранять** указанные обозначения **либо** действовать по отдельному письменному соглашению (white-label).

## 4. Обзор продукта

Надстройка для **WPS Writer** (**Vue 3 + Vite**): чат ИИ, суммаризация, анализ текста, перевод, мультимодальность, безопасность и рассекречивание, пакетная обработка документа/таблиц/рисунков, формы, шаблоны и правила, список задач и оркестрация — с записью в документ (**вставка / замена / комментарий / связанный комментарий / добавление в конец**).

**Приоритет офлайн / интранет:** **Ollama** и **OpenAI-совместимые** шлюзы (LM Studio, Xinference, OneAPI, …).

---

## 4.1 Версия 2.0.0: крупный рефакторинг и стабильность

**Текущая версия: `2.0.0`.** Релиз обобщает и переносит в README результаты Markdown-документов по планированию и реализации: план v2, P0-P6, Workflow W1-W7, переработка задач, закрытие runtime-разрывов и форма настройки ассистентов.

- Ассистенты с параметрами перед запуском показывают форму: целевой язык для перевода; соотношение сторон, длительность и стиль голоса для изображений, видео и аудио.
- Выбор моделей стал типизированным: в чате видны только диалоговые модели; настройки группируют и фильтруют модели по типам.
- Офлайн / интранет-развертывание через Ollama, LM Studio, Xinference, OneAPI, New API и OpenAI-совместимые шлюзы остается приоритетом.
- Улучшены стабильность задач, повторы, передача параметров, окно прогресса и запись результата в документ.

Полное сравнение с конкурентами по **60 пунктам** находится в основном README: [README.md § 4.2](README.md#42-60-项竞品能力对比察元-200-vs-常见办公-ai--文档-ai-工具).

## 6. Карта функций

**О Chayuan**, **ассистент Chayuan AI**, **анализ текста**, **перевод**, **мультимодальность**, **умные ассистенты** (меню «Ещё» и контекстное меню), **безопасность**, пакетные операции с **документом / таблицами / изображениями**, **Chayuan AI Review** (формы, аудит, шаблоны, правила), **настройки**, контекстное меню **добавить в ассистент Chayuan AI**.

**Расширение:** пользовательские ассистенты, оркестрация задач, режим отчёта.

---

## 7. Встроенные ассистенты (кратко)

Проверка орфографии и грамматики (JSON), резюме, перевод, текст→изображение/аудио/видео; переписывание, расширение, сокращение; пояснения к комментариям и гиперссылкам; исправление орфографии; ключевые слова; проверка нумерации абзацев; проверка «следов ИИ»; проверка конфиденциальности; извлечение секретных ключевых слов; извлечение полей форм; аудит документа; стилизация (полировка, официальный стиль, упрощение); действия и риски; унификация терминов; заголовки; структура абзацев; протокол совещания; стиль политических/официальных документов. Подробные описания и промпты см. в исходниках (`assistantRegistry.js`).

**Пользовательские ассистенты:** свои сценарии отчётов, аннотаций и правок.
