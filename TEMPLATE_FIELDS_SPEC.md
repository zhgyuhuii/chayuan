# 规则制作 - 表单项规则字段说明

## 标签格式

插入到文档的标签（书签）名称为：`书签名称|id`

- 示例：`姓名|f001`、`合同金额|f002`
- 便于根据书签名称解析出书签名称和规则 id，从而在审查时匹配对应规则

---

## 字段列表

### 1. 书签名称（必填）

**说明**：该项在表单中的显示名称，将作为标签的一部分。

**填写建议**：
- 简短清晰，2–8 个字为宜
- 使用中文或常见业务用语

**示例**：`姓名`、`合同编号`、`签署日期`、`合同金额`

---

### 2. 是否必填

**说明**：标记该项是否为必填。必填项在审查时会校验是否为空。

**填写建议**：
- 核心字段（如姓名、日期）建议勾选
- 可选说明性字段可不勾选

---

### 3. 数据类型与约束（选择后动态显示）

选择数据类型后，表单会动态显示该类型专属的约束项。

| 类型 | 约束项 | 说明 |
|------|--------|------|
| **字符串** | 是否必填、最小长度、最大长度、须包含、不许包含、正则匹配 | 长度与内容约束 |
| **整数** | 是否必填、最小值（≥）、最大值（≤）、等于（精确值） | 数值范围 |
| **小数** | 是否必填、最小值、最大值、等于、小数位数 | 数值范围与精度 |
| **日期** | 是否必填、不早于（≥）、不晚于（≤）、等于 | 日期范围 |
| **时间** | 是否必填、不早于、不晚于、等于 | 时间范围，格式 HH:mm |
| **日期+时间** | 是否必填、不早于、不晚于、等于 | 日期时间范围 |
| **布尔** | 是否必填、允许的值 | 如：是,否（逗号分隔） |
| **邮箱/电话/身份证/网址** | 是否必填 | 格式由类型自动校验 |

---

### 4. 审查方式

**说明**：选择对该项内容的审查方式，与类型约束配合使用。

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 无 | 不做额外审查 | 仅用类型约束即可 |
| 正则审查 | 用正则表达式校验格式 | 固定格式（身份证、手机号等） |
| 格式校验 | 按数据类型做格式校验 | 日期、邮箱、数字等 |
| 范围校验 | 数值或长度在指定范围内 | 金额区间、字数限制 |
| 枚举校验 | 只能是指定选项之一 | 固定选项（是/否、同意/不同意等） |
| 大模型审查 | 用 AI 判断内容是否符合描述 | 语义、合理性检查 |
| 敏感词审查 | 检查是否包含敏感词 | 禁止出现特定词汇 |
| 格式一致性 | 与文档中其他同类项格式一致 | 多处填写需统一 |
| 逻辑关系 | 与关联项满足逻辑关系 | 如结束日期晚于开始日期 |
| 跨项校验 | 与指定书签内容一致或关联 | 多处内容需一致 |

### 5. 审查提示

**说明**：校验不通过时向用户展示的提示语，便于填写者修正。

**示例**：`请输入2-4个汉字`、`金额需在0-100万之间`

---

### 6. 审查规则

**说明**：与审查方式配合使用，定义具体校验规则。

#### 6.1 正则审查

- **填写方式**：输入正则表达式
- **示例**：
  - 手机号：`^1[3-9]\d{9}$`
  - 身份证：`^\d{15}|\d{18}$`
  - 2–4 个汉字：`^[\u4e00-\u9fa5]{2,4}$`
  - 日期 YYYY-MM-DD：`^\d{4}-\d{2}-\d{2}$`

#### 6.2 格式校验

- **填写方式**：通常留空，使用数据类型的默认格式
- 如需自定义，可填如：`YYYY-MM-DD`、`YYYY/MM/DD`

#### 6.3 范围校验

- **填写方式**：`最小,最大`（英文逗号分隔）
- **示例**：
  - 金额 0–100 万：`0,1000000`
  - 字数 1–200：`1,200`（针对字符串长度）

#### 6.4 枚举校验

- **填写方式**：多个选项用英文逗号分隔
- **示例**：
  - `是,否`
  - `同意,不同意,弃权`
  - `甲方,乙方`

#### 6.5 大模型审查

- **填写方式**：简要描述审查标准或期望
- **示例**：
  - `判断是否为合理的中文人名`
  - `判断金额是否为合理的合同金额`
  - `判断日期是否在合理范围内`

#### 6.6 敏感词审查

- **填写方式**：不允许出现的词，多个用逗号分隔
- **示例**：`绝密,机密,内部`

#### 6.7 格式一致性

- **填写方式**：留空则与同类型项比对
- **说明**：确保与文档中其他同类项格式一致

#### 6.8 逻辑关系

- **填写方式**：描述与其他书签的逻辑关系
- **示例**：`endDate>startDate` 表示结束日期晚于开始日期

#### 6.9 跨项校验

- **填写方式**：填写关联的书签 id
- **示例**：`f001` 表示需与该项一致

---

### 7. 备注

**说明**：可选，用于团队内部说明该项用途、来源或约定。

**示例**：`从合同正文提取`、`需与附件一致`

---

## 规则与标签的对应关系

1. **插入标签**：点击操作列对号或**双击数据行**，在光标处插入书签，名称为 `书签名称|id|序号`（如 `姓名|f001|1`），序号自动递增避免重复
2. **保存规则**：规则列表中的每一项对应一个唯一 `id`
3. **审查时**：
   - 遍历文档中的书签
   - 解析书签名得到 `书签名称` 和 `id`
   - 用 `id` 查找对应规则
   - 按规则中的审查类型和审查规则校验书签内的内容

## 扩展字段（后续可增加）

| 字段 | 说明 |
|------|------|
| 默认值 | 未填写时的默认内容 |
| 占位提示 | 在输入框中的占位文案 |
| 最大/最小长度 | 字符串长度限制 |
| 日期格式 | 自定义日期显示格式 |
| 错误提示 | 校验失败时展示的提示语 |
| 显示顺序 | 表单项的排序 |
| 分组 | 对表单项分组归类 |
