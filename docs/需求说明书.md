# 察元 AI 助理 / 察元 AI 编审 — 详细需求说明书

> 基于历史提示词与对话记录整理  
> 项目：niudang（WPS 插件）  
> 技术栈：Vue 3 + Vite + WPS JS API

---

## 一、项目概述

### 1.1 项目定位

察元 AI 助理 / 察元 AI 编审是一款基于 WPS 的 AI 办公插件，为文档编辑提供智能辅助能力，包括 AI 对话、翻译、摘要、文本分析、拼写语法检查、文档审查与批注、表单编辑等。

### 1.2 核心能力模块

| 模块 | 功能描述 |
|------|----------|
| AI 助手 | 对话、翻译、摘要、文本分析、文本转图像/语音/视频 |
| 拼写与语法检查 | 文档拼写与语法错误检测与修正 |
| 模型配置与管理 | 多模型供应商、API 配置、模型分组与开关 |
| 表单编辑与书签 | 书签列表、表单编辑、文档定位 |
| 文档审查与批注 | 文档审查、批注插入、AI 痕迹检查（已移除） |

### 1.3 技术架构

- **前端**：Vue 3、Vite
- **宿主**：WPS 桌面客户端
- **API**：WPS JS API、各类 AI 模型 API
- **数据持久化**：localStorage + 文件 + PluginStorage 多层存储

## 二、助手设置需求

### 2.1 默认设置精简

- **保留项**：仅保留以下 5 项
  - 对话模型
  - 图像生成模型
  - 视频生成模型
  - 语音转换模型
  - 段落截取设置

### 2.2 模型继承规则

- 未单独指定模型的助手（拼写、摘要、翻译等）**自动使用默认对话模型**
- 助手按模型类型分类：
  - 分析型 → 对话模型
  - 图像型 → 图像模型
  - 视频型 → 视频模型
  - 语音型 → 语音模型

### 2.3 助手类型与模型筛选

| 助手类型 | 可选模型范围 |
|----------|--------------|
| 文本转图像 | 仅图像模型 |
| 文本转视频 | 仅视频模型 |
| 文本转语音 | 仅语音模型 |

### 2.4 媒体助手真实调用

- **文本转图像**：调用 `/images/generations` 接口，生成后插入文档
- **文本转语音**：走独立接口
- **文本转视频**：走独立接口

### 2.5 显示位置配置

- 支持配置项：
  - 顶部主菜单
  - 更多菜单
  - 右键菜单
  - 右键更多菜单
- **主菜单与更多菜单互斥**：只能选其一

### 2.6 显示优先级

- 支持 `displayOrder` 排序
- 支持上移/下移、拖拽排序
- 上移/下移按钮使用单箭头图标（上箭头、下箭头）
- 移除左侧拖拽手柄，助手图标置左

### 2.7 菜单位置预览

- 实时预览助手在各菜单中的落位
- **后续计划**：移除该预览功能

### 2.8 图标配置

- 从离线图标库选择图标
- 新建助手有默认图标
- 顶部「更多」菜单与设置面板中助手图标保持一致

### 2.9 助手列表

需全量落地的助手类型：

- 拼写检查
- 生成摘要
- 文本分析
- 翻译
- 文本转图像
- 文本转语音
- 文本转视频
- 创建智能助手

## 三、模型设置需求

### 3.1 模型供应商管理

- **添加入口**：模型清单底部「添加」行，点击弹出对话框
- **配置项**：图标、名称、类型
- **供应商类型**：OpenAI、OpenAI-Response、Gemini、Anthropic、Azure OpenAI、New API、CherryIN、Ollama 等
- **自定义图标**：支持上传自定义图标（非固定图标）
- **右键菜单**：仅用户添加的模型支持删除、编辑

### 3.2 模型清单交互

- 支持拖拽排序，带插入线提示
- 模型分组可折叠
- 分组名称和子项采用扁平化样式
- 模型列表支持开关，关闭的模型不在 AI 助理中显示

### 3.3 API 配置

- **获取秘钥链接**：在「多个密钥使用逗号分隔」后增加「获取秘钥」链接，按模型打开对应平台
- **链接打开方式**：使用 `OAAssist.ShellExecute` 在系统浏览器中打开
- **无 API URL 的提示**：对 Ollama 等无云端 API 的模型，显示「查看 XXX 文档和模型获取更多详情」
- **秘钥链接修正**：修正龙猫、MiniMax、百川等 API 秘钥链接
- **龙猫 404 修复**：默认 API 地址改为 `https://api.longcat.chat/openai`

### 3.4 界面优化

- 移除粘贴/清空按钮
- 检测按钮改为图标并带悬停提示
- API 秘钥和 API 地址输入框内支持清空图标
- 默认未选中模型时，默认选中 OpenAI

### 3.5 模型类型显示

- 在管理模型和模型列表中显示：嵌入、图像、视频等类型

### 3.6 默认设置布局

- 默认模型改为「默认设置」
- 中间为模型设置项，右侧为设置
- 默认模型使用分组下拉，带图标，按类型过滤
- 分组可折叠

### 3.7 段落截取设置

- 截取长度
- 上下重叠长度
- 分段策略：按段落 / 按句子 / 按字符

### 3.8 百度云千帆

- 替换文心一言为百度云千帆
- 修复 API 调用
- 使用蓝橙双色 3D 六棱柱图标

### 3.9 移除项

- 顶部菜单移除模型选择下拉
- 移除 AI 痕迹检查按钮
- 移除文档检查按钮

## 六、AI 助手对话框需求

### 6.1 布局

- 左侧：历史对话

- 模型选择、输入框、附件、发送按钮同一行（单行输入区）

- 模型下拉被遮挡时改为向上展开

### 6.4 对话能力

- 流式输出，实时展示对话内容
- 插入文档/批注：图标形式，在输出右下角，支持插入前确认

### 6.5 右键预填

- 右键「添加到察元AI助手」后，选中内容自动填入输入框

### 6.6 错误处理

- 402 错误为 API 余额不足，非 WPS 限制

### 6.7 数据持久化

- 使用 localStorage + 文件，避免 PluginStorage 不持久

## 十、技术实现要点

### 10.1 核心工具模块

| 模块 | 职责 |
|------|------|
| `globalSettings.js` | 全局设置，文件 + localStorage + PluginStorage 多层存储 |
| `modelTypeUtils.js` | 模型类型区分：chat、embedding、image、video、voice |
| `mediaApi.js` | 图像、语音、视频生成独立处理 |
| `documentActions.js` | 替换、插入、批注等文档动作统一处理 |
| `assistantTaskRunner.js` | 各类助手任务统一执行 |
| `assistantRegistry.js` | 助手注册与管理 |
| `assistantSettings.js` | 助手配置 |
| `modelSettings.js` | 模型配置 |
| `modelGroups.js` | 模型分组 |
| `chunkSettings.js` | 段落截取设置 |

### 10.2 WPS 兼容

- 外部链接使用 `OAAssist.ShellExecute` 在系统浏览器打开
- ShowDialog 中粘贴使用 Clipboard API

### 10.3 主要组件

| 组件 | 职责 |
|------|------|
| `SettingsDialog.vue` | 设置主界面 |
| `AIAssistantDialog.vue` | AI 助手对话界面 |
| `ribbon.js` | 顶部 Ribbon 与菜单 |
| `Popup.vue` | 弹窗组件 |
| `FormEditDialog.vue` | 表单编辑 |
| `public/ribbon.xml` | Ribbon 配置 |
