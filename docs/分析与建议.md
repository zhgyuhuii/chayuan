## 2026-03 当前实现更新（第三版）

经过三轮完整补齐，7 大链路已从"主骨架可用"升级为"治理层全面生效"。在前两轮基础上，第三轮重点补齐了 5 个平台级能力：

- `capabilityPolicyStore.js`：执行前策略门禁，风险分级（inferCapabilityRiskLevel）、高风险确认拦截、策略快照，已集成到 capabilityBus 和 wpsCapabilityExecutor。
- `documentOperationLedger.js`：逐操作级账本，支持 replace/insert-after/comment/append/prepend 等动作记录，设置页可预览/选择重放/导出。
- `chatMemoryStore.js` + `buildContextBudgetPlan`：长期记忆按文档作用域存取，动态预算策略（light/standard/tight 三档），摘要质量抽检标记。
- `buildAssistantHealthScore` + `buildAssistantReleaseGate`：健康分和发布门禁，已嵌入创建/修复/进化/晋升全路径。
- `buildEvaluationDashboard`：评测看板，场景/样本/Owner 聚合，趋势统计。

加上前两轮已有的 9 个关键平台层：

- `artifactTypes.js` / `artifactStore.js` / `artifactRenderer.js`：统一文件生成与识别产物协议。
- `multimodalRecognitionRunner.js`：图片理解、ASR 转写 + 音频理解双层流水线、视频动态采样 + 分段理解。
- `capabilityBus.js`：WPS 原生能力从"零散执行器"升级为"可注册、可编排、策略门禁、审计导出"的能力总线。

### 仍未完全满分的地方

- `普通聊天`：摘要仍为规则型压缩，缺独立的人工抽检/复核面板。

- `原生能力支持`：设置页缺策略配置 UI，业务图中的超大能力池仍可扩展。
- `文件生成/多模态识别`：`useServerFallback` 只打了标志无真实调用链，缺 OCR 独立通道和镜头级理解。

- `修复助手 / 助手进化`：缺批量回归、自动双跑和完整评测工作台。

7. Excel / Markdown / JSON 文件生成建议
这是当前项目非常适合补齐的一块。

Excel
不要让模型直接“生成 xlsx 二进制”，而是先生成工作簿结构：

{
  "sheets": [
    {
      "name": "问题清单",
      "columns": ["序号", "问题", "风险级别", "建议"],
      "rows": [
        [1, "合同主体缺失", "高", "补全主体名称"]
      ]
    }
  ]
}
再由本地 renderer 用 xlsx 落成真实文件。
你项目里已经有 xlsx 依赖，这条很适合快速补上。

Markdown
最容易落地，建议统一支持：

摘要
审计报告
会议纪要
清单类结果
JSON
JSON 应该成为所有复杂产物的“中间协议”，不是只当调试输出。
